阅读须知

本文只做转正述职的部分内容总结记录，具体工作内容不会详细去说，在之前 7 月份总结的时候就发现有小伙伴觉得工作太简单了，还有提及答辩我应该怎么怎么去说等等。

有好的建议挺好的，但本文内容与正式转正述职内容还是有些许差距，仅做参考哈。

第一份工作

首先，我还是想说我来到 CODING，是比较幸运的。

在选择 CODING 之前，在杭州有一段实习经历，所以 CODING 应该算是我的第一份正式工作。不得不说这也是一场缘分吧，来讲讲与 CODING 的邂逅。

邂逅 CODING

在大三阶段我其实就有了解到这个网站，从何了解呢？

当初大三就折腾过搭建 Hexo 博客，当时代码是放在 GitHub 托管的，网站部署也是使用 GitHub Pages 服务，但访问速度很慢。

后来通过网上的博客教程了解到可以通过托管到 CODING 代码仓库，然后通过静态托管就可以加速访问速度，犹记得当时还是从腾讯云某个入口进入的。

而开始那会为了找静态托管入口也是找了许久，因为教程中就有那个选项，而我打开却没有。后面才知道需要在设置里面开启才能使用。

当时托管我记得还需要支付小额的资金，就是因为这个付费操作，我记住了这个网站，当时还挺不可思议的，怎么之前都没怎么听说。

最开始在校我了解到代码托管的就是国外的 GitHub，国内的 Gitee，而当时就了解到了 CODING，也是最开始与 CODING 的邂逅，后面就来到了 CODING。

在 CODING 的感悟

接下来，我就来说说来到 CODING 后的一些感悟：

首先，还是学习了许多关于公司的文化方面，在封培期间，通过前辈们的讲述与参与活动，让我对 CODING 文化理解更加深刻。

在实习期间，那次季度总结会议上听了张老板的讲述 CODING 的一些未来愿景与使命，也激发了我的一些热情，让我觉得虽然目前所做的事情只是一小步，但是未来也许是一大步。

我在做一件很有意义的事情，甚至我想说我在改变世界。

甚至我想说我在改变世界。

所以总结来说，征途是星辰大海。

在过去大学里面，我对于技术方面也是比较热爱，但现在来到 CODING 之后，我发现，诶，我不仅 Love Coding, 我还 Love CODING。

也是在前辈们的讲述以及自己这段时间的工作，让我感受到了 CODING 技术上的沉淀，当初选择也是考虑想要去一家以技术为主要驱动的公司，来做有意义的事情，也许选择会比努力更重要。

对未来的展望：

这里我就以做 CRM 工作为例子讲了，比如在做功能模块页面开发以及将涉及到模块的页面进行重构的时候，其中就遇到过一些业务与技术上的问题，当时与导师还有产品进行了沟通，导师给予了一些比较好的技术方案，不管是技术广度还是深度方面都很优秀。

在解决这些问题之后，让我对技术方面也有了更多的热爱，追求更好的代码质量以及前端美学，把产品做得美观也是吸引客户的一个要素。

因此，在今后的工作日子中，我会主动思考与总结，遇到问题去解决，主动担当，然后保持热爱，追求卓越。

对公司的建议和意见

这里我就以校招生的角度来说一说：也许提不出特别好的建议，但更多的也是一份祝愿吧。

希望能够增强团队的影响力，突显团队优势。

怎么说呢，在过去大学求职期间，就有听说过许多公司的团队，虽然我不清楚团队内的成员是哪些，但是当听到这个团队名称的时候，就能感受到技术上的影响力，比如当初前端方面，就有了解到腾讯文档团队，有赞的前端团队等。

所以希望的是对于 CODING，在未来的发展中，比如当提到研发效能、战略发展，某某团队呀，就能联想到 CODING，就是 No.1。
